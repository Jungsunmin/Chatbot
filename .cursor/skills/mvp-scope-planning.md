# MVP Scope Planning

## When to use

- 기능 목록이 많을 때 **Must vs 2차** 강제 분리
- 일정·리소스 협상 전

## Prerequisites

- `requirements-decomposition` 초안 또는 사용자 기능 리스트

## Procedure

1. Must = 1.0 출시 최소
2. Should = 1.x 직후
3. Could = 2차(커뮤니티·SSO 등)
4. Non-Scope = 명시적 제외(모더레이션 없는 커뮤니티, 포털 대체 등)
5. Release Criteria 체크리스트
6. **사용자 승인** 후 DECISIONS에 기록

## Output template

`docs/agentic/skill-outputs/mvp-scope-planning.md`

```md
# MVP Scope
## Must

## Non-Scope
## Release Criteria
```
