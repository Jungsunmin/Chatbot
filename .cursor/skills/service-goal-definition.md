# Service Goal Definition

## When to use

- 신규 기능·제품 방향 정의
- `context_packet`의 Confirmed Goal이 TBD이거나 변경될 때

## Prerequisites

- [`docs/agentic/context_packet.md`](../../docs/agentic/context_packet.md)
- [`.cursor/skills/skill-contract.md`](skill-contract.md)

## Procedure

1. 문제·페인포인트를 **검증 가능한 문장**으로 정리
2. 1차/2차 사용자 구분
3. 가치 제안(Why this app vs 기존 채널)
4. 성공 기준 3~5개(측정 가능하면 수치 초안)
5. Non-Goals·리스크 명시
6. 사용자 승인 요청 후 문서 반영

## Output template

`docs/agentic/skill_outputs.md` → `## service_goal_definition`

```md
# Service Goal Definition
## Problem
## Target Users
## Value Proposition
## Success Criteria
## Non-Goals
## Risks
```
