## service_goal_definition

# Service Goal Definition

## Problem

건국대학교(KU) **외국인 유학생**은 입학 전·재학 중 **국제처·외국인학생센터** 및 분산된 학교 안내(가이드북, 포털, 공지)에 접근하기 어렵다. 합격 직후 필요한 **서류·비자·학사 일정·기숙사·캠퍼스 생활** 정보가 한곳에 정리되어 있지 않아, 입국 전 준비와 등록·수강신청 등 핵심 시점에서 **누락·지연**이 발생할 수 있다.

## Target Users

| 구분 | 설명 |
|------|------|
| **1차** | KU에 입학·등록 예정 또는 진행 중인 **외국인 유학생**(학부·대학원·어학당·교환 포함) |
| **2차** | 국제처·외국인학생센터 **행정 담당자**(공식 콘텐츠·공지 연동·검수 — MVP 이후 본격) |
| **3차** | 재학 외국인 **동료 지원**(커뮤니티·룸메이트 — MVP 범위는 제한) |

**온보딩 상태**: 입학 전 → 재학(학사) → 생활(캠퍼스·기숙사 등)에 따라 홈(대시보드) 경험을 분기한다.

## Value Proposition

- **다국어(영·중·일)** 로 KU 가이드북 핵심 섹션을 **앱 단일 진입점**으로 제공한다.
- **상태별 맞춤 대시보드**로 “지금 단계에서 해야 할 일”을 보여 준다.
- **학사·비자 등 주요 일정 Push**, **캠퍼스 지도**, **FAQ(검색·AI 보조)**, **수강신청 가이드**, **통합 알림·커뮤니티**로 건국대형 **통합 유학생 앱** 경험을 지향한다.

## Success Criteria

| ID | 기준 | 측정(초안) |
|----|------|------------|
| SC-1 | 입학 전 사용자가 **가이드북 핵심 체크리스트**를 앱에서 완료 가능 | 온보딩 완료율, 체크리스트 완료 수 |
| SC-2 | **3개 언어** UI 전환 시 핵심 화면·가이드북 본문 누락 없음 | i18n QA 통과율, 번역 커버리지 |
| SC-3 | 수강신청·비자 연장 등 **주요 일정 알림**이 기한 전에 도달 | 알림 클릭률, 일정 놓침 관련 CS 감소(장기) |
| SC-4 | FAQ·가이드로 **국제처 단순 문의** 일부 대체 | FAQ 검색·해결률, 담당자 피드백(협업 후) |

## Non-Goals (서비스 전체 관점)

- 타 대학·타 캠퍼스 범용 플랫폼
- 학교 **공식 성적·등록 시스템 전체 대체**(포털 SSO·성적 API 없이는 조회만 연동 또는 2차)
- **비공식·출처 불명** 행정 안내의 단독 게시(공식 가이드북·국제처 검수 우선)
- MVP 단계에서 **국제처 공식 승인·운영 위임** 전제로 한 전면 자동화

## Risks

| 리스크 | 설명 |
|--------|------|
| R-1 | 가이드북·일정 정보 **오래됨** → 잘못된 준비 |
| R-2 | AI FAQ **환각·오답** → 행정 피해 |
| R-3 | 커뮤니티·룸메이트 **개인정보·안전** |
| R-4 | 지도·길찾기 **데이터 부정확** |
| R-5 | Push·알림 **권한 거부·플랫폼 정책** |
| R-6 | MVP **비공식 개발** → 브랜드·콘텐츠 검수 지연 |

## User Scenarios

1. **입학 전**: 합격 후 앱 설치 → 언어 선택 → “입학 전” 대시보드 → 가이드북 비자·서류 체크 → D-7 비자 마감 Push.
2. **첫 학기**: “재학” 전환 → 수강신청 도우미 스크린샷 따라하기 → 수강신청 일정 알림.
3. **생활**: 캠퍼스 지도로 국제처 건물 검색 → FAQ “dormitory deposit” 검색 → AI가 가이드북 절 연결.
4. **커뮤니티**: 국적 그룹에 질문 → (2차) 룸메이트 매칭 요청.
5. **알림 센터**: 학교 공지·행사 한 목록에서 확인.

## Edge Cases

- 사용자가 **복수 프로그램**(교환+학부)에 해당하는 경우 → 프로그램 태그·복수 체크리스트.
- 가이드북 **섹션만 번역 완료** → 폴백 언어 + “번역 준비 중” 배지.
- Push **거부** → 앱 내 알림 센터·배지로 대체.
- AI **근거 없음** → 검색 결과만 + 국제처 연락처.
- 커뮤니티 **신고·차단** 미구현 시 → MVP에서 게시 제한 또는 읽기 전용.

## Acceptance Criteria (통합)

- AC-1: 3언어 전환 후 홈·가이드북·설정 **크래시 없음**.
- AC-2: 온보딩 상태 3종 전환 시 홈 **카드 구성이 달라짐**.
- AC-3: MVP Must 섹션 가이드북 **3언어 노출**.
- AC-4: 등록된 일정 1건 이상 **Push 수신** 검증(테스트 환경).
- AC-5: FAQ 검색 **키워드 매칭**; AI는 **출처 표시** 또는 FAQ-only 모드.
- AC-6: 지도에서 **건물 1건 검색·상세** 표시.

## mvp-scope-planning

# MVP Scope

## Must (MVP 1.0 — 출시 최소)

| 영역 | 포함 내용 |
|------|-----------|
| **다국어** | EN/ZH/JA 전환, 핵심 화면·가이드북 MVP 섹션 |
| **대시보드** | 입학 전 / 재학 / 생활 **3상태 분기** 홈(카드·다음 할 일) |
| **가이드북** | KU 가이드북 **우선순위 섹션** 이식(입학·비자·등록·기숙사 등), 3언어 |
| **일정·알림** | 주요 일정 템플릿 + **Push**(수강신청·비자 연장 최소 2종) |
| **FAQ** | 키워드 검색 + **FAQ-only 또는 RAG 제한** 답변(출처 필수) |
| **알림 센터** | 공지·행사 **수동 등록 피드**(국제처 협업 전 CMS/JSON) |
| **플랫폼** | iOS + Android |
| **인프라** | 백엔드 API + 콘텐츠 저장(버전·언어 필드), 인증(이메일/게스트 중 택1) |

**가이드북 MVP 섹션(제안 — 확정 필요)**: Welcome, Visa/Immigration, Enrollment, Housing, Academic Calendar, Emergency Contacts.

## Should (MVP 1.x — 직후 스프린트)

| 영역 | 포함 내용 |
|------|-----------|
| **캠퍼스 지도** | 건물 검색 + 주요 기관 카드; 길찾기는 단순 경로 또는 외부 맵 링크 |
| **수강신청 도우미** | 단계별 스크린샷 가이드(3언어) |
| **가이드북** | 전 섹션 확장, 오프라인 캐시 |
| **일정** | 사용자 맞춤 일정 추가, 캘린더 연동(ics export) |
| **국제처 협업** | 콘텐츠 검수 워크플로, 공식 공지 연동 |

## Could (2차)

| 영역 | 포함 내용 |
|------|-----------|
| **커뮤니티** | 국적별 그룹 게시판 |
| **룸메이트 매칭** | 프로필·매칭 |
| **지도** | 실시간 길찾기·실내 층 |
| **포털 SSO** | 학번 로그인, 성적·공지 API |
| **AI** | 대화형 확장, 다국어 STT |

## Non-Scope (MVP)

- 타 대학 지원
- **공식 성적·수강신청 시스템** 대체
- **전체 커뮤니티·룸메이트** (신고·모더레이션 없이 오픈)
- **국제처 공식 운영·브랜딩 승인** 전제의 대외 홍보
- 무제한 **외부 LLM** 자유 채팅(출처 없는 행정 답변)

- [ ] TestFlight/내부 테스트 또는 소규모 파일럿(20+ 유학생) 2주
